1.11. Проверка рабочего процесса 

Сделайте форк личного проекта. Если в разделе личного проекта нет ссылки на репозиторий, то надо немного подождать, пока наш робот её создаст.

Затем клонируйте форк личного проекта себе на компьютер.

В форке личного проекта создайте рабочую ветку.

В этой ветке внесите изменения в файл Readme.md: укажите имя вашего наставника.

Установите зависимости, выполнив команду npm install.

Чтобы проверить работу приложения, выполните команду npm start.

Перейдите по адресу, указанному в терминале (скорее всего, это будет http://localhost:5173/). Если сборка прошла успешно, то на странице вашего приложения вы увидите Hello, world!.

Сделайте коммиты ваших изменений локально, а затем синхронизируйте локальный и удалённый репозитории (ваш форк).

Создайте пулреквест из рабочей ветки вашего форка в мастер-репозиторий и прикрепите задание с помощью кнопки «Найти пулреквест», расположенной над этим заданием в блоке деталей проекта.

Для проверки задания отправьте ссылку на пулреквест в чат вашей группы. Не торопитесь нажимать кнопку «Принять задание». Это можно будет сделать после проверки задания на видеовстрече.
